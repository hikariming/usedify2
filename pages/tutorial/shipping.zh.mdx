title: "自信上线"
---

最后一步聚焦于生产级上线的准备工作。

## 可观测性清单

- [ ] 结构化日志，确保 Trace ID 贯穿 Dify 回调全链路。
- [ ] 将关键指标（延迟、成本、满意度）输出到 Prometheus 或 OpenTelemetry。
- [ ] 配置 Slack/Teams 实时告警，及时发现错误率波动。

## 安全与合规

1. 每 30 天轮换一次 Dify API Key。
2. 使用白名单与范围受限的凭证控制工具权限。
3. 对对话记录进行静态加密，并自动脱敏个人隐私数据。

## 发布手册

| 阶段 | 负责人 | 行动 |
| ---- | ---- | ---- |
| 演练 | 技术负责人 | 在预发布环境回放真实对话，校验回复质量 |
| 金丝雀 | SRE | 放量至 5% 流量，实时监控仪表盘 |
| 全量上线 | 产品负责人 | 对外公告并持续收集反馈 |

为每一次事故建立复盘机制，将经验沉淀回生命周期管理中，持续提升质量。
