最後のステップでは、本番運用の準備に集中します。

## 可観測性チェックリスト

- [ ] 構造化ログを有効にし、トレース ID を Dify の Webhook へ伝播させる。
- [ ] レイテンシ、コスト、満足度のメトリクスを Prometheus または OpenTelemetry にエクスポートする。
- [ ] エラーレート上昇を Slack / Teams でリアルタイム通知する。

## セキュリティとコンプライアンス

1. Dify の API キーを 30 日ごとにローテーションする。
2. 許可リストとスコープ付き認証情報でツール権限を制限する。
3. 会話ログを保存するときは保存時暗号化と自動 PII マスキングを適用する。

## ローンチプレイブック

| フェーズ | オーナー | アクション |
| ----- | ----- | ------- |
| ドライラン | テックリード | 実際の会話をステージングで再生し、応答を検証する |
| カナリア | SRE | トラフィックの 5% にリリースし、ダッシュボードを監視する |
| 一般提供 | プロダクト | ユーザーへ告知し、フィードバックループをモニタリングする |

すべてのインシデントを記録し、得られた知見をライフサイクルに還元して品質向上を継続しましょう。
