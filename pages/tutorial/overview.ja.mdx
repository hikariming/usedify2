**From Zero to Dify Agent** チュートリアルへようこそ。このロードマップでは、各モジュールがどのように積み重なっていくのかを示し、学習の道筋を立てやすくします。

## このチュートリアルで作るもの

最終的には、次の特徴を備えた本番対応のエージェントワークスペースを出荷します。

1. Dify のワークフローオーケストレーションと統合し、ツール呼び出しエージェントを実装する。
2. データセット・フィードバックシグナル・ガードレールを活用した評価ループを構築する。
3. API ゲートウェイの背後にデプロイし、利用分析とコストコントロールを備える。

## モジュール構成

| モジュール | フォーカス | 成果 |
| ------ | ----- | -------- |
| 環境構築 | ツールチェーン、リポジトリ、品質ゲート | ローカルとクラウドで再現可能な開発環境 |
| ライフサイクル | プロジェクトの足場とドメイン設計 | チーム向けの意見主導アーキテクチャ |
| 本番出荷 | 可観測性、スケーリング、コンプライアンス | ユーザーへのリリースに自信を持てる状態 |

サイドバーから最初のレッスンに移動するか、前提条件を確認しましょう。

## 前提条件

- TypeScript / Python の基礎に慣れていること。
- API 認証情報を持つ Dify インスタンス（クラウドまたは自前ホスト）へのアクセス。
- Git ワークフローに関する基本的な理解。
