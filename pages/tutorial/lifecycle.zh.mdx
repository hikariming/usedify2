title: "规划智能体生命周期"
---

import { Callout } from 'nextra/components';

优秀的 Dify 智能体需要兼顾试验速度与可靠性。以下生命周期可帮助团队保持一致。

## 1. 发现（Discover）

- 访谈业务方，明确成功指标。
- 在协同白板（如 Miro）梳理用户旅程与上下文窗口。
- 制定北极星场景与「首个可用版本」目标。

## 2. 原型（Prototype）

- 在 Dify 创建沙箱数据集，生成合成对话数据。
- 通过特性开关（如 LaunchDarkly）逐步开放功能。
- 使用 `.prompt` 文件将提示词纳入版本控制。

## 3. 评估（Evaluate）

- 将 Dify 评估任务接入 CI，自动回归测试。
- 覆盖对抗性提示与工具失败场景。
- 在仪表盘（Metabase、Grafana 等）可视化漂移并设置告警。

## 4. 上线与学习（Launch & Learn）

- 采用金丝雀发布策略逐步推向生产环境。
- 从 Dify 会话记录中收集结构化反馈。

<Callout>
生命周期是循环往复的：每次上线都会反哺下一轮「发现」阶段。
</Callout>
